Payment API Integration Guide

This document explains how to integrate the payment API with your application. The process involves sending transaction details, generating a unique transaction code and signature, preparing a payload, and finally redirecting to the eSewa payment page.

1. Sending Transaction Details

Users must provide the following fields:

  • amount (mandatory)
  • product_code (mandatory)
  • delivery_charge
  • service_charge
  • tax_amount
Error Response: If amount or product_code are missing, the following error will be returned: 
{
  "code": "ES705",
  "message": "Invalid transaction amount."
}

2. Backend Transaction Processing

On the backend:

  • A transaction_uuid is generated using uuid.uuid4().
  • A signature is created based on the required fields.
  • A payload is constructed and returned to the user.

Example Payload

{
  "total_amount": "100.00",
  "transaction_uuid": "f22f91a6-afcc-4df7-97ce-4a99b124a399",
  "product_code": "EPAYTEST",
  "amount": "100.00",
  "tax_amount": "0.00",
  "product_service_charge": "0.00",
  "product_delivery_charge": "0.00",
  "success_url": "http://localhost:8000/success/",
  "failure_url": "http://localhost:8000/fail/",
  "signed_field_names": "total_amount,transaction_uuid,product_code",
  "signature": "4Mggg5jAsEXLz2ttU6NRmvJ9knq5g1BNyrfqft/G6OY="
}

3. Submitting the Payload

The payload data should be submitted as a POST form to the eSewa test endpoint:

<form method="POST" action="https://rc-epay.esewa.com.np/api/epay/main/v2/form">  
  <input type="text" name="amount" value="100.00">
  <input type="text" name="product_code" value="EPAYTEST">
  <input type="text" name="transaction_uuid" value="f22f91a6-afcc-4df7-97ce-4a99b124a399">
  <input type="text" name="signature" value="4Mggg5jAsEXLz2ttU6NRmvJ9knq5g1BNyrfqft/G6OY=">
  <input type="text" name="success_url" value="http://localhost:8000/success/">
  <input type="text" name="failure_url" value="http://localhost:8000/fail/">
  <!-- Add other required fields from the payload -->
  <button type="submit">Pay Now</button>
</form>

4. Redirect to Payment Page

Once the form is submitted, the user is automatically redirected to the eSewa payment page where they can complete the transaction.

Note: Make sure you are using the test endpoint (rc-epay) during development. Replace it with the live endpoint when moving to production.
